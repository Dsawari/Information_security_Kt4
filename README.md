# Information_security_Kt4
Социальная инженерия и защита от фишинга.

Задание 1. Идентификация фишинга.
1. Первое сообщение с высокой вероятностью является фишинговым, тк присуствует подозрительный URL (домен не совпадает с реальными доменами банков), используется HTTP вместо защищенного HTTPS и присуствуют подозрительные слова по типу "secure" и "auth".
2. Второе сообщение можно считать легитимным, тк присутсвует персонализация по имени и отчеству, а как же ссылка на гос портал HTTPS.
3. Третье сообщение является фишинговым, тк используется поддельный домен, а так же присутсвует эмоциональное давление для того, чтобы пользователь точно перешел по ссылке.

Задание 2. Кейсы
Кейс 1: "Срочный платеж".
1. Какая именно техника социальной инженерии использовалась?
- Первая техника это авторитет, те мошенник представляется генеральным директором. Вторая это срочность, а третье это создание стрессоваой ситуации, указано, что ген.директор «на совещании» и не может ответить, тем самым исключается возможность перепроверки.
2. Какие две основные угрозы скрываются в действии Марины (открытие архива)?
- Во-первых, риск заражения системы вредоносным ПО при открытии архива, поскольку внутри может находиться исполняемый файл или скрипт, и, во-вторых, риск компрометации финансовых данных компании, если файл содержит поддельные реквизиты для перевода средств злоумышленникам.
3. Какой единственный и обязательный шаг должна была сделать Марина перед открытием файла?
- Единственный обязательный шаг, который должна была сделать Марина перед открытием файла, это верификация запроса по независимому каналу связи. Универсальное правило для сотрудников: перед выполнением любых срочных финансовых операций или открытием вложений от руководства необходимо подтвердить подлинность запроса через альтернативный канал.

Кейс 2: «Звонок из банка»
1. Какие две разные техники социальной инженерии были скомбинированы в этой атаке? Как злоумышленники добились того, что SMS пришло с официального номера банка?
- Первое - вишинг (обман по телефону с представлением сотрудником службы безопасности банка), второе - использование одноразового кода подтверждения для атаки с перехватом аутентификации.
2. В чём заключаются две ключевые поведенческие ошибки Ивана?
- Первая ошибка заключается в том, что Иван доверился звонку без проверки личности. А вторая ошибка - он сообщил код из смс.
3. Что Иван должен был сделать вместо диктовки кода?
- Иван долден был прекратить разговор и не сообщать код. Далее стоит связаться с банком по оф номеру и проверить инфоомацию, если нужно, то заблокировать карту.

Кейс 3: «Подарок от коллеги»
1. Почему эта атака считается более опасной, чем массовый фишинг? Как злоумышленники могли получить доступ к аккаунту Максима в чате?
- Она считается более опасной, чем массовый фишинг, потому что происходит внутри корпоративного канала и от имени знакомого коллеги, что снижает уровень подозрительности.
2. Какие три красных флажка (подозрительные детали) должен был заметить Антон в полученном сообщении?
- Во-первых, странное название адреса (yadi.sk). Во-вторых, наличие пароля для доступа к файлу, это странно с точки зрения защиты, тк пароль передается в том же сообщении. В-третьих, имя файла «architecture_draft.pdf.exe», тут явно не пытаются отправить pdf файл, а хотят чтобы мы запустили исполняемы файл exe.
3. Предложите для компании практическое правило на уровне политики ИБ, которое позволило бы предотвратить такие инциденты.
- Все файлы должны передаваться исключительно через утверждённые корпоративные хранилища, а запуск исполняемых файлов запрещён без проверки службой информационной безопасности. Также обязательным должно быть использование двухфакторной аутентификации для доступа к корпоративным сервисам.
